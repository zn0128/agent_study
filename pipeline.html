AI Agent · Pipeline 流程图

以"帮我写一份 AI Agent 架构分析报告"为例 · 复杂任务路径（Plan-and-Execute）
INPUT 👤 用户输入 message · sessionId · apiKey → POST /api/chat
01 🛡️ Guardrails 正则扫描 · 破坏性操作 / 提示词注入 / 角色覆盖 输入护栏
→ ⛔ 命中规则 → 直接拦截，不消耗 API
↓ safe = True
02 💾 Memory 三层检索 L1: session_facts (RAM) · L2+3: TF-IDF cosine → memories.json 持久化库 + 向量索引
↓ session_facts[] + vector_hits[]（带相关度分数）
03 ✍️ Prompt 构建 角色定义 + 工具规则 8 条 + 注入记忆 → sys_prompt
04 📄 Context 统计 估算 token 占用 · 组装 messages[] · 截取最近 8 轮历史
05 💭 Planning 关键词匹配：分析/报告/制定/规划/总结/研究/帮我写
简单任务 · CoT / ReAct
⤷ 直接进入 LLM 主循环
复杂任务 · Plan-and-Execute
06 🔄 Workflow（本地） 提取话题 → 知识检索 → 新闻判断
↓ wf_context（预检索结果）
07 🎯 Orchestrator 流式调度 SubAgent · 逐步 yield 事件
LLM① 🧠 decompose_task() Claude API · 生成子任务列表 ~3s
LLM② 🧠 SubAgent #1 独立 Context + 专项系统提示 · 执行子任务 ~5s
LLM③ 🧠 SubAgent #2 独立 Context + 专项系统提示 · 执行子任务 ~5s
注入 messages：SubAgent 结果[:800] + Workflow 上下文[:400] + 指令
⟳ LLM 主循环 · ReAct · 最多 5 轮
LLM④ 🧠 主 Agent 推理 claude-haiku-4-5 · max_tokens=2048（复杂）/ 1024
stop_reason = tool_use → 进入工具链（右侧）→ 结果回传，继续推理
stop_reason = end_turn → 退出循环
stop_reason = max_tokens → 收集所有 text 块，退出循环
↩ tool_result 追加到 cursor，re-enter LLM（iter_i + 1）
final_text 写入，进入下一阶段
工具调用链（tool_use 时触发）
①
👤 HITL memory write → 审核
②
🔌 MCP 协议路由 + 调用日志
③
📚 RAG knowledge_base / memory_bank
④
🔧 Tools calculate / datetime / news…
⑤
⭐ Skills knowledge_integration / news_digest…
🔒 save_to_memory / add_to_memory_bank
触发 HITL · await asyncio.Event()
POST /api/hitl/{id}/approve or reject
08 🛡️ Guardrails 扫描输出：API Key 泄露 / 密码信息 输出护栏
→ 命中 → 替换为提示文本
09 💾 Memory 更新 auto_extract_session() → 提取姓名等事实 → session_facts
10 📊 Evaluation base 0.72 + 工具 +0.10 + 长度 +0.10 + 格式 +0.03 ≤ 0.99
OUTPUT 👤 用户输出 SSE event: text → 浏览器渲染 Markdown
模块类型： 核心（LLM/Prompt/Planning） 记忆知识（Memory/RAG） 行动（MCP/Tools/Skills） 执行（Workflow/SubAgent） 可靠性（Guardrails/HITL/Eval） LLM 调用（Claude API）
